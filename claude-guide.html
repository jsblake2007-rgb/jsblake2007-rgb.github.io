01 — Overview

What is Claude?

Claude is an AI assistant made by Anthropic. It understands and generates text, reads files, writes and debugs code, analyses documents, and holds long, coherent conversations. The underlying models are named in the Claude 4 family (Opus, Sonnet, Haiku), each trading off speed vs. capability.

🕮
Claude Opus 4
The most powerful model. Best for complex reasoning, long documents, nuanced writing, and hard coding tasks. Slower and uses more credits.
⚡
Claude Sonnet 4
The balanced workhorse. Fast, smart, and cost-efficient. The default for most everyday tasks — coding, drafting, research, Q&A.
🚀
Claude Haiku 4
The lightest model. Near-instant replies, great for quick lookups, simple edits, or building apps where latency matters.

03 — Prompting

Writing Great Prompts

The biggest difference between a mediocre result and a great one is almost always the prompt. Claude is literal — the more precisely you describe what you want, the better it performs.

1
Give context first
Tell Claude who you are, what you're building, and why before asking your question. Context shapes the answer.
I'm a junior developer building a personal portfolio site in plain HTML/CSS. Explain how CSS Grid works in simple terms and give me a 3-column card layout example.
2
Specify the format you want
Ask for bullet points, numbered steps, a table, code with comments, or a short paragraph. Claude matches whatever format you request.
Give me a comparison of React vs Vue in a markdown table with columns: Learning Curve, Performance, Ecosystem, Best For.
3
Constrain length and tone
Add phrases like "in under 150 words", "be concise", "explain like I'm 15", or "professional tone" to get exactly what you need.
4
Provide examples (few-shot prompting)
Show Claude a sample of what "good" looks like. It will match the pattern for your actual request.
Convert these function names to snake_case. Examples: getUserData → get_user_data, fetchItems → fetch_items. Now convert: calculateTotalPrice, sendEmailNotification, parseUserInput.
5
Iterate — don't start over
Claude remembers the whole conversation. Follow up with "make it shorter", "use TypeScript instead", or "now add error handling" — no need to repeat yourself.
6
Use Projects for ongoing work
(Pro only) A Project on claude.ai gives Claude persistent memory of your codebase, style guide, or background info across every conversation in that project. Set it up once with a system prompt, then every chat already knows your context.
04 — Claude Code

Claude Code — The CLI

Claude Code is a command-line tool that puts Claude directly inside your terminal (or IDE). It reads your actual files, runs commands, edits code, and commits to git — no copy-pasting required. This very guide was created with it.

1
Install
Requires Node.js 18+. Install globally with npm:
npm install -g @anthropic-ai/claude-code
2
Start a session
Navigate to your project folder and run claude. The first run will ask you to log in with your Anthropic/Claude.ai account.
cd my-project
claude
3
Talk to it naturally
Just describe what you want. Claude Code can read, write, search, and run things in your repo. You approve anything that looks risky before it runs.
Add a dark-mode toggle to my index.html using CSS variables and a small JS button in the nav.
4
Key slash commands
/help — show all commands
/clear — clear the conversation context
/review — review the current git diff
/compact — summarise conversation to save context
/config — open settings (model, theme, etc.)
5
Use a CLAUDE.md file
Put a CLAUDE.md file in the root of your project. Claude Code reads it automatically at the start of every session — put your stack info, coding conventions, and preferred patterns there so you never have to explain them again.
6
IDE integration
Claude Code has official extensions for VS Code and JetBrains. Search "Claude Code" in the extensions marketplace. This opens a side panel so you never leave your editor.
05 — Best Practices

Pro Tips

  • Pick the right model for the job. Use Sonnet for everyday tasks — it's fast and smart. Save Opus for genuinely hard problems (complex architecture, long document analysis). This keeps your usage limits healthy.
  • Context window is precious. Very long conversations degrade quality as older messages scroll out. Use /compact or start a fresh conversation when a topic is done.
  • Be specific about constraints. "Write a function that does X" is weaker than "Write a pure JavaScript function (no dependencies) that does X, handles null input, and includes a JSDoc comment."
  • Use Projects for recurring work. (Pro) Create a Project for each ongoing codebase or client. Add a system prompt describing the stack and your preferences. Every conversation inherits that context automatically.
  • Upload your files. Claude can read PDFs, images, code files, CSV data, and more. Paste or upload the source material — don't try to summarise it yourself.
  • Ask Claude to verify itself. After a complex answer, follow up with "Are you confident in this? What could go wrong?" — Claude is good at self-critique and will flag uncertainties honestly.
  • Don't accept the first draft. Claude excels at iteration. Ask it to "make this more concise", "add edge-case handling", or "rewrite in a more conversational tone" — each pass is usually significantly better.
  • For code: provide the error message. Paste the exact stack trace or error output. Claude diagnoses from real errors far more accurately than from descriptions of errors.
  • Trust but verify. Claude is very capable but not infallible. Always review generated code before running it in production, and check factual claims on anything high-stakes.
  • Claude Code: stay in the repo. Claude Code works best when you're in the project directory. It reads your actual files rather than relying on descriptions — the less you have to explain, the better the results.
